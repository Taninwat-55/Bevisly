# Bevisly v0.2.0 Roadmap

The **v0.1.0 MVP** of Bevisly establishes the core user flows for proven,
skill-based hiring. As we move towards **v0.2.0**, our focus shifts securely
scaling the platform, improving data integrity, and polishing the user
experience (UX) to enterprise standards.

---

## 🔒 1. Security & Data Integrity (Priority)

Before driving significant traffic to the platform, we must ensure all user data
is strictly isolated and secure against malicious input.

- [ ] **Strict Row Level Security (RLS) Audit:**
  - Verify that `invitations`, `user_goals`, `companies`, and `company_members`
    have watertight policies.
  - Ensure candidates cannot read other candidates' submissions or private
    feedback.
  - Ensure employers can only access submissions and feedback related to their
    own jobs/company.
- [ ] **Secure Public Profiles:**
  - Enforce `is_public = true` at the database RLS level for the `profiles`
    table. The API should return `0` rows for unauthorized queries to private
    profiles, rather than relying on the frontend to filter them out.
- [ ] **Admin Data Viewer Lockdown:**
  - Audit `AdminDataViewer.tsx` and the `/admin` routing to guarantee that
    absolutely no PII or sensitive data is accessible to non-admin roles through
    API interception.
- [ ] **Input Sanitization (XSS Protection):**
  - Implement a robust HTML sanitizer (e.g., `DOMPurify`) on the frontend
    anywhere we render Markdown or AI-generated HTML (especially the
    `JobDetailPage.tsx` descriptions).

---

## ✨ 2. UX Enhancements & Polish

Polishing the rough edges of the MVP to reduce friction and improve the
"time-to-value" for end users.

- [ ] **SME Slide-Over Panels (Drawers):**
  - Convert the Candidate submission review and grading interfaces on the
    Employer Dashboard from full-page navigations into Side-Drawers/Slide-overs.
  - **Benefit:** Allows employers to rapidly review dozens of candidates
    sequentially without losing their place on the Talent Board.
- [ ] **Actionable Empty States:**
  - Implement beautiful, engaging empty states across the application.
  - Candidate Dashboard: "Find your first task" CTA when no active workflows
    exist.
  - Employer Dashboard: "Post your first role" CTA when the talent board is
    empty.

---

## 🚀 3. Growth & Acquisition

Leveraging our unique AI tools to drive organic sign-ups.

- [ ] **Landing Page Tool Integration:**
  - Bring the "Generate Proof Task" AI feature directly to the public landing
    page (`LandingTool.tsx`).
  - Allow prospective employers to input a basic job title and instantly receive
    a sample proof task _before_ they are prompted to create an account.
  - **Benefit:** Immediate demonstration of Bevisly's core value proposition
    without the friction of a signup wall.
